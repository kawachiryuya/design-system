## 設計思想：4層構造と依存ルール

Atomic Design の「依存方向の制約」を継承しつつ、分類の曖昧さを排除した構造です。

- **Primitives**：単一のHTML要素をラップした最小部品。他のコンポーネントを import しない。
- **Composites**：Primitives を組み合わせた複合コンポーネント。Primitives のみ import 可。
- **Patterns**：Composites を組み合わせたセクション単位のパターン。Composites + Primitives を import 可。
- **プロダクト層**：すべてを import 可。デザインシステム外で管理。

```
Primitives    他のコンポーネントを import しない
    ↑
Composites    Primitives のみ import 可
    ↑
Patterns      Composites + Primitives を import 可
    ↑
プロダクト層   すべてを import 可（デザインシステム外）
```

Patterns は「複数プロダクトで再利用される汎用パターン」のみ採用し、ビジネスロジックを含みません。
プロダクト固有のコンポーネントは **[Product名]** カテゴリとして同じ Storybook 上で確認できます。

## デザイントークン

すべてのコンポーネントは `tokens/` ディレクトリの JSON ファイルを参照しています。
Tailwind CSS の設定にも統合済みです。

- **Colors** — プライマリ・ニュートラル・セマンティックカラー
- **Typography** — フォントサイズ・ウェイト・行間・フォントファミリー
- **Spacing** — 8px ベースのスペーシングスケール
- **Shadows** — none〜2xl のエレベーション（box-shadow）
- **Radius** — none〜full の角丸（border-radius）
- **Animation** — デュレーション・イージング関数

→ サイドバーの **Tokens** カテゴリで視覚的に確認できます。

## アクセシビリティへの取り組み

すべてのコンポーネントで以下を考慮しています。

- **ARIA属性**：role・aria-label・aria-invalid などを適切に付与
- **キーボード操作**：Tab・Enter・矢印キーによる操作対応
- **色コントラスト**：WCAG 2.1 AA 基準（4.5:1以上）を遵守
- **a11y アドオン**：各Storyの「Accessibility」タブで検証結果を確認できます
