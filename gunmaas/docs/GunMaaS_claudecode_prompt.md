# GunMaaS リブランディング — 実装移植の指示

## 背景

GunMaaS（群馬版MaaS）のリブランディングプロジェクトで、Claude.ai上での壁打ちを通じてUI/UX設計を進めてきました。現在、以下の成果物があります：

1. `GunMaaS_unified_prototype.jsx` — 全4タブ（ホーム・検索・チケット・マイページ）＋購入フロー＋行き先詳細ページを含むインタラクティブなプロトタイプ
2. `GunMaaS_design_decisions.md` — 12の設計判断の経緯と理由
3. `GunMaaS_claude_code_instructions.md` — 画面別の実装方針・ファイル構成案・データモデル

以前のClaude Codeセッションで、デザインシステム（https://github.com/kawachiryuya/design-system）のトークンやコンポーネントをある程度構築しています。ただし、その後Claude.aiでの設計議論でUI構造が大きく変わったため、**実装上の古い内容と最新のプロトタイプの間にズレがあります**。

## やってほしいこと

### 1. 現状の把握
まず以下を確認してください：
- デザインシステムリポジトリの現在の状態（トークン・コンポーネントの実装状況）
- `GunMaaS_unified_prototype.jsx` の全体構造
- `GunMaaS_design_decisions.md` の設計判断
- `GunMaaS_claude_code_instructions.md` の実装方針

### 2. 差分の整理
プロトタイプと既存の実装を比較して、以下を整理してください：
- 既存実装から**そのまま使えるもの**（トークン定義、基本コンポーネント等）
- **更新が必要なもの**（プロトタイプで変わった部分）
- **新規で作る必要があるもの**

### 3. 実装の移植
プロトタイプを参照しながら、デザインシステムのトークンを使った本格実装に移植してください。

**重要な方針：**
- プロトタイプはあくまで「設計の参照」であり、コードをそのまま使うのではなく、デザインシステムのトークン・コンポーネント構造に合わせて再実装する
- GunMaaS固有のカラーパレット（#2D6A4F等）は維持しつつ、デザインシステムのsemantic colors構造に合わせる
- 1ファイルにすべてが入ったプロトタイプを、適切なコンポーネント分割・ファイル構成に整理する

### 4. 優先順位
1. ホームタブ（マップ＋検索バー＋チップス＋ボトムシート）
2. 検索タブ（経路検索＋ルート詳細＋チケット提案）
3. 購入フロー（4ステップ）
4. 行き先詳細ページ（3タブ）
5. チケットタブ（利用中/過去）
6. マイページ（オンボーディング＋アカウント管理）
