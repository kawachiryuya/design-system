# GunMaaS リブランディング — 設計判断サマリー

## プロジェクト概要

GunMaaS（https://g3m.jp/）— 群馬県新モビリティサービス推進協議会が運営する地域型MaaSプラットフォームのフルスコープ・リブランディング（VI・UI・IA・ブランド戦略）。

---

## 重要な設計判断の経緯

### 1. タスク指向 → OOUI（オブジェクト指向UI）への転換

**判断:** 「行き先（Destination）」をメインオブジェクトとして設計の中心に据える。

**理由:** 3ペルソナ（観光客・通勤者・高齢者）のJTBDを横断すると共通するのは「どこかに行きたい」という意図。チケットもアクセスも予約も、行き先オブジェクトのプロパティとして整理することで、ユーザーの思考モデルに合致する。

### 2. 3セグメント → セグメント分岐なし

**経緯:**
- 初期: 3セグメント（観光/暮らし/高齢者）
- Phase 3: 2セグメント（旅する/暮らす）— 高齢者セグメントを廃止（「自分の力で外出したい」というJTBDと矛盾）
- 最終: セグメント分岐なし — OOUI転換により、行き先オブジェクトのカテゴリ（観光/駅・バス停/施設）がセグメントの役割を吸収

**理由:** 旅でも日常でも目的地を探す行為は同じ。ホームタブで分岐点を置くのはノイズ。

### 3. タブバー4構成

| タブ | 役割 | 設計方針 |
|------|------|----------|
| ホーム | 行き先の発見（What） | マップ＋チップス＋ボトムシート |
| 検索 | 経路検索＋チケット提案（How） | 出発地・目的地・日時のフル入力 |
| チケット | 購入済みチケットホルダー | 利用中/過去の切替。ブラウジング機能は置かない（OOUI的にチケットは行き先経由で発見される） |
| マイページ | アカウント・認証管理 | 会員QR＋プログレッシブ・オンボーディング |

### 4. ホームタブのUI構造（Apple Maps型）

**判断:** マップを常時背景として表示し、検索バーとボトムシートをオーバーレイする。

**理由:** マップメインかリストメインかの二項対立ではなく、「マップが常にそこにあって、シートを引き上げればリストコンテンツが見える」レイヤー構造。リスト⇔マップの切替を不要にした。

**検索バー:**
- 「どこに行く？」の1フィールド入力
- タップで入力モード → サジェストパネル展開（履歴＋人気の行き先＋リアルタイム検索結果）
- 結果タップ → 検索タブに目的地プリセットで遷移（指名検索は行き先詳細を経由しない）
- 右に「経路を検索」ボタン → 検索タブに遷移

**カテゴリチップス:** 「すべて / 観光 / 駅・バス停 / 施設」の3+1カテゴリ。マップ上のピンとボトムシートのリストが連動してフィルタリング。

**レイヤーコントロール:** マップ右下に1つのレイヤーアイコン。タップでパネル展開 → 鉄道路線・デマンド交通エリアを個別ON/OFF。

### 5. ホーム検索バー vs 検索タブ検索バーの棲み分け

- **ホーム:** 「どこに行く？」1フィールド。目的が曖昧な状態の入口。
- **検索タブ:** 出発地（デフォルト「現在地」）＋目的地＋日時のフル入力。目的が明確な状態の実行ツール。

### 6. 経路検索のルートカード設計

**チケット情報の表示:**
- 一覧段階でチケット名＋価格＋性質を並記（「🎫 草津高原線フリーパス ¥1,540 / バス区間2日間乗り放題」）
- 「おトクなチケットあり」だけでは不十分。ユーザーが一覧段階でお得度合いを判断できる情報を出す
- 正確な割引後金額は出せない（フリーパスは期間型、回数券は累積型）ため、チケットの価格と性質を事実として提示

**ルート詳細（ボトムシート）:**
- タイムライン表示（出発地=グリーン●大、途中駅=路線色●、目的地=テラコッタ●大）
- 目的地名が行き先オブジェクトの場合はリンク化（タップで行き先詳細へ）
- チケット提案セクション（ルート全体に対するおすすめを比較）
- 割引バナー（条件付き表示：マイナンバー未登録＋割引対象の場合のみ）

### 7. コンテキストカード（デマンド交通補完）

**判断:** デマンド交通は経路検索エンジンがGTFS-Flex未対応のため拾えない。UIレイヤーで補完する。

**実装:** 検索結果一覧の最上部にコンテキストカードとして表示（「このエリアでは呼べば来るバスが使えます」＋予約ボタン）。まともなルートが出ない場合は警告メッセージも表示。

### 8. 行き先オブジェクトの3タブ構成

| タブ | 内容 |
|------|------|
| 基本情報 | 住所・電話番号・営業時間のみ。アクセス情報は削除（経路検索と重複）|
| チケット | 使えるチケット一覧＋予約・手配情報（統合）|
| 周辺 | 周辺スポット |

**CTA「この行き先へ経路検索」:** ヒーロー（グリーン背景）内に常時表示。タブの切替状態に関わらず常にアクセス可能。タップで検索タブに目的地プリセットで遷移。

### 9. 購入フロー（4ステップ）

1. **チケット概要** — 名前・説明・金額。「ログインして購入する」（未ログイン時）or「購入手続きへ」（ログイン済み）
2. **認証** — JRE IDログイン/新規登録（未ログイン時のみ）
3. **購入確認** — チケット情報＋マイナンバー割引バナー（条件付き・インライン登録可能）＋決済方法選択（クレカ/Apple Pay）＋利用ICカード確認
4. **完了** — 「ルート詳細に戻る」＋「チケットを確認する」（→チケットタブ遷移）

**遷移方式:** ルート詳細のボトムシートを閉じてから、購入フローを全画面ページとして表示（モーダルの中にモーダルを避ける）。

**マイナンバー割引:** 購入確認画面（ステップ3）で「今すぐ登録して割引を適用」→ インラインでマイナンバー読み取りUI → 完了後に金額が即座に更新。

### 10. プログレッシブ・オンボーディング

**3段階:**
1. 登録なしで使える（経路検索・行き先閲覧・マップ探索）
2. チケット購入時にJRE ID登録を促す
3. 割引利用時にマイナンバーカード登録を促す

**マイページの表示:**
- 未完了時: 「GunMaaSをもっと便利に使おう」＋説明文＋プログレスバー
- 全完了時: 「登録済みの情報」＋「✓ すべて完了」（説明文非表示）

### 11. マイページの構成

- **プロフィールカード:** 会員QRコード＋名前＋ID（ゲスト時はアバター）
- **セットアップ:** 3ステップカード（JRE ID / ICカード / マイナンバーカード）、展開可能で関連操作をインライン表示
- **その他の登録:** こどもデマンド利用登録
- **サポート:** はじめ方ガイド / FAQ / お問い合わせ
- **その他:** 利用規約 / プライバシーポリシー / 運営情報
- **ログアウト**

**削除した項目:** 設定セクション（通知・言語・位置情報 — Webサービスのため不要）、購入履歴（チケットタブに集約）、割引ステータス（購入時に表示されるため重複）

### 12. チケットタブの設計

- **購入済み特化:** ブラウジング機能は置かない
- **利用中:** チケットカード（ヘッダーバンド＋パンチ穴切り取り線＋有効期限＋残り日数）＋「チケットを表示する」＋「取り消し・払い戻し」
- **過去:** 薄い表示＋「もう一度購入」＋「領収書」

## ブランド定義

- **コアバリュー:** 「群馬のすべての移動をつなぎ、あなたの『行きたい』をかなえる。」
- **タグライン:** 「群馬を、つながる。」
- **パーソナリティ:** 親しみやすさ・頼れる安心感・群馬らしさ・シンプルさ
- **トンマナ:** 行政用語を避け日常語で（「デマンド交通」→「呼べば来るバス」）

## 3ペルソナ × ユーザーフロー

### 美月さん（観光客30代）
ホームのマップで草津温泉を発見 → ピンタップ → 「ここへ行く」→ 検索タブ（目的地プリセット）→ 経路検索 → ルート詳細 → チケット提案 → 草津高原線フリーパス購入

### 健太さん（通勤40代）
検索タブを直接開く → 前橋→高崎を検索 → ルートカードでチケット価格を確認 → 中心市街地乗り放題券を購入 → マイナンバー割引を購入確認画面で適用

### 幸子さん（高齢者72歳）
ホームのマップでデマンドゾーンを確認 → ゾーンタップ → 「予約する」でデマンド予約。経路検索は使わないルート。

- 多言語対応: 観光MaaSとしての将来拡張
