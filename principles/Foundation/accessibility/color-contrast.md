## 原則の定義

**テキストとUI要素は、背景との間に十分なコントラスト比を保つ**

## WCAG基準

### 最小コントラスト比

| 要素タイプ | AA（推奨） | AAA（理想） |
|-----------|-----------|------------|
| 通常テキスト（14-18px） | 4.5:1 | 7:1 |
| 大きなテキスト（18px以上） | 3:1 | 4.5:1 |
| UI要素（ボタン境界線等） | 3:1 | - |

### 推奨する組み合わせ

```
✅ 良い例:
- 黒テキスト（#1a1a1a）on 白背景: 15.3:1
- 濃いグレー（#4a4a4a）on 白背景: 8.6:1

❌ 悪い例:
- グレー（#9ca3af）on 白背景: 2.8:1 ← 基準未満
- 青（#60a5fa）on 白背景（小テキスト）: 2.9:1 ← 基準未満
```

## 色だけに依存しない

### 原則

状態や情報を色だけで表現しない。アイコン、テキスト、パターンを併用。

```
✅ 良い例: 色 + アイコン + テキスト
<div class="error">
  <ErrorIcon />
  <span>エラー: 入力内容を確認してください</span>
</div>

❌ 悪い例: 色のみ
<div class="text-red">入力内容を確認してください</div>
```

## 色覚異常への配慮

### 主な色覚異常のタイプ

- **1型（赤）**: 約1%（男性）
- **2型（緑）**: 約5%（男性）
- **3型（青）**: 稀

### 推奨事項

- 赤と緑の組み合わせを避ける
- Chrome DevToolsで色覚シミュレーション
- 複数の視覚的手がかりを使用

## ツール

- Chrome DevTools: Lighthouseスコア
- [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)
