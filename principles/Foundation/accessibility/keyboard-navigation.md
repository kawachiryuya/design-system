## 原則の定義

**すべてのインタラクティブ要素は、マウスなしでキーボードのみで操作できる必要がある**

##基本キー操作

| キー | 動作 |
|------|------|
| Tab | 次の要素へフォーカス移動 |
| Shift + Tab | 前の要素へフォーカス移動 |
| Enter | リンク/ボタンを活性化 |
| Space | ボタン/チェックボックスを活性化 |
| Escape | モーダル/ドロップダウンを閉じる |
| ↑↓←→ | リスト/タブ/メニュー内移動 |

## フォーカス順序

### 原則

- **論理的な順序**: 視覚順序とDOM順序を一致
- **tabindex="0"**: 自然なTab順序に含める
- **tabindex="-1"**: プログラムでのみフォーカス可能
- **tabindex="1以上"**: 使用禁止（予測不能）

### 良い例・悪い例

```
✅ 良い例: 自然なDOM順序
<form>
  <input />
  <button>送信</button>
</form>

❌ 悪い例: tabindexで順序制御
<button tabIndex={2}>送信</button>
<input tabIndex={1} />
```

## カスタムコンポーネント

**推奨**: ネイティブ要素（`<button>`等）を優先使用

カスタム要素を使う場合は、適切なroleとキーボードハンドラーが必要。

→ 実装詳細は各コンポーネントドキュメント参照
