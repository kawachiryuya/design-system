## 原則の定義

**タッチ可能な要素は、誤タップを防ぎ、正確な操作を可能にする十分なサイズを確保する**

## 最小サイズ

### WCAG 2.1基準

| プラットフォーム | 最小サイズ | 基準 |
|----------------|-----------|------|
| モバイル | 44px × 44px | WCAG 2.1 AAA (2.5.5) |
| デスクトップ | 40px × 40px | 推奨 |

### 設計根拠

- 平均的な指のサイズ: 約40-50px
- 精密なタップが困難なユーザーへの配慮
- 移動中の操作を考慮

## 実装方法

### paddingで調整

視覚的に小さくても、タップ領域を確保。

```css
.small-icon-button {
  /* 視覚: 16px icon */
  width: 16px;
  height: 16px;
  /* タップ領域: 44px */
  padding: 14px;
}
```

### 最小サイズの適用

```
Button Small: 視覚32px + padding 12px = 44px ✅
Icon Button: アイコン24px + padding 10px = 44px ✅
Link: 本文16px + padding 14px縦 = 44px ✅
```

## 間隔

### 要件

- **最小間隔**: 8px
- **推奨間隔**: 12-16px

### 理由

隣接する要素を誤タップしないため。

## 特殊ケース

### 密なUIが必要な場合

データテーブル、ツールバー等で密なUIが必要な場合：

- デスクトップのみで使用
- または、拡大モード提供
- モバイルでは別UI提供

## 測定方法

### チェックリスト

- [ ] すべてのボタン・リンクが44px×44px以上
- [ ] アイコンボタンにpaddingで領域確保
- [ ] 隣接要素との間隔が8px以上

### テスト

- モバイル実機でタップテスト
- 親指で到達可能か確認
