## 原則の定義

**アイコンは言語を補完するものであり、単独で意味を伝えることを期待しない**

アイコンは視覚的な手がかりとして機能しますが、文化・年齢・経験によって解釈が異なります。テキストと組み合わせることで、誰にでも正確に意味が伝わります。

## なぜアイコンが必要か

- **スキャンの高速化**: テキストより早く認識できる
- **スペース効率**: 限られた空間に情報を収める
- **言語を超えた理解**: 直感的なビジュアルコミュニケーション
- **視覚的なリズム**: リストやナビゲーションに統一感を与える

## アイコン使用の3原則

### 1. テキストと組み合わせる

```
❌ 悪い例: アイコンのみ（ツールバー）
          [🖊] [🗑] [📋] [⚙]
          → 何のボタンか分からないユーザーがいる

✅ 良い例: アイコン + ラベル
          [🖊 編集] [🗑 削除] [📋 コピー] [⚙ 設定]

例外: 極めて普遍的なアイコン（✕の閉じるボタン等）は
      aria-labelで補完すればアイコン単独も可
```

### 2. 一貫したスタイルを使う

同一アプリ内で複数のスタイルを混在させない。

```
✅ 統一: すべてOutlineスタイル
❌ 混在: あるページはOutline、別のページはFilled
```

→ 詳細は [Styles](./styles.md) 参照

### 3. 意味に対応したアイコンを使う

慣習から外れたアイコンを使わない。

```
✅ 慣習通り: 🏠 = ホーム、✉ = メール、⚙ = 設定
❌ 独自解釈: ⚡ = 設定（ユーザーが混乱する）
```

→ 詳細は [Semantic Icons](./semantic-icons.md) 参照
