# アイコンスタイル

## メタ情報
- カテゴリ: コンテンツ原則 > Iconography
- 適用範囲: Atom〜Organism
- ステータス: Approved
- 最終更新: 2024-02-17

## 原則の定義

**システム内でアイコンのスタイルを統一し、一貫した視覚言語を維持する**

## スタイルの種類

### Outline（アウトライン）

輪郭線だけで描かれたアイコン。軽量で控えめな印象。

```
☆ ♡ 🔔（線のみ）

用途:
- 非アクティブな状態（ナビゲーション・タブ）
- Secondary / Tertiary要素
- 情報的なアイコン（ヘルプ・情報）
```

### Filled（フィルド）

塗りつぶされたアイコン。重みがあり、存在感が強い。

```
★ ♥ 🔔（塗りつぶし）

用途:
- アクティブな状態
- 重要なアクション
- エラー・警告・成功の状態アイコン
```

### Outline → Filled で「状態」を表現

```
非アクティブ → アクティブ:
☆ ナビ項目    →    ★ ナビ項目（選択中）
♡ お気に入り  →    ♥ お気に入り（登録済み）
```

この使い分けにより、アニメーションや色の変化なしでも状態変化が伝わる。

## システム内での統一ルール

### 1つのライブラリに絞る

```
✅ 推奨: 1つのアイコンライブラリを選んで統一
  例: Heroicons, Lucide, Material Icons いずれか1つ

❌ 非推奨: 複数ライブラリの混在
  Heroicons + Font Awesome + 独自SVG が混在
  → スタイルが揃わず視覚的に散漫
```

### Outlineを基本とする

```
デフォルト: Outline
アクティブ・重要・状態表現: Filled

理由: Outlineが基本だと、Filledが「特別な意味」を持つことが
      明確になり、状態変化がより伝わりやすくなる
```

## 色との組み合わせ

アイコンの色は、隣接するテキストまたはコンテキストに合わせる。

```
標準テキストと並ぶ場合: neutral-700（テキストと同色）
Primaryアクション: brand-600
Success: success-600
Error: error-600
Warning: warning-600
Disabled: neutral-400
```

→ 詳細は [Semantic Colors](../../color/semantic-colors.md) 参照

## ストロークの太さ（Outline）

アイコンライブラリによってストローク幅が異なる。システム内で統一する。

```
推奨: 1.5px〜2px（細すぎず太すぎず）
小サイズ(12px)では2pxが視認性が良い
大サイズ(32px以上)では1.5pxが洗練された印象
```

## チェックリスト

- [ ] アイコンライブラリが1つに統一されている
- [ ] Outlineを基本、Filledをアクティブ/状態に使っている
- [ ] アイコンの色がコンテキストに合っている
- [ ] 非アクティブ→アクティブの切り替えがOutline→Filledで表現されている
