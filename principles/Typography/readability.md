# 可読性（Readability）の原則

## メタ情報
- カテゴリ: コンテンツ原則
- 適用範囲: Molecule〜Page
- ステータス: Approved
- 最終更新: 2024-01-15

## 原則の定義

**テキストコンテンツを最適な形式で提示し、ユーザーがストレスなく読み進められるようにする**

可読性は、フォントサイズや行間だけでなく、行長、段落間隔、余白、コントラストなど、複数の要素が組み合わさって実現されます。優れた可読性により、ユーザーは情報を素早く理解し、長時間の読書でも疲れにくくなります。

## 背景と目的

### なぜ可読性が重要か

- **理解速度の向上**: 読みやすいテキストは内容の理解が早い
- **疲労の軽減**: 適切な設定により、目の疲労を最小化
- **離脱率の低下**: 読みにくいテキストはユーザーを離脱させる
- **アクセシビリティ**: ディスレクシア、弱視などのユーザーにも配慮
- **コンバージョン向上**: 読みやすいコンテンツは行動を促す

## 行長（Measure / Line Length）

### 最適な文字数

読みやすい1行の文字数には科学的根拠があります。

**推奨範囲**:
- **英語**: 45-75文字（理想は66文字）
- **日本語**: 25-35文字

### 設計根拠

- **短すぎる**: 視線が頻繁に折り返し、リズムが途切れる
- **長すぎる**: 次の行頭を見つけにくく、読み飛ばしが発生
- **理想的**: 自然な視線の移動で快適に読める

### 実装方法

```
英語コンテンツ:
<div class="max-w-prose">  <!-- 65ch -->
  <p>Long English text...</p>
</div>

日本語コンテンツ:
<div class="max-w-[35ch]">
  <p>日本語の長文...</p>
</div>

記事本文（より狭く）:
<div class="max-w-2xl">  <!-- 672px -->
  <article>...</article>
</div>
```

### コンテキスト別の行長

| コンテンツタイプ | 推奨最大幅 | 理由 |
|---------------|-----------|------|
| 記事本文 | 65-75ch (英) / 35ch (日) | 長時間の読書に最適 |
| リード文 | 75ch (英) / 40ch (日) | やや広めでゆったり |
| UIテキスト | 制限なし | 機能優先 |
| モバイル | 画面幅の90% | 余白とのバランス |

## 段落間隔

### 基本ルール

**WCAG 2.1推奨**: 段落間は行間の2倍

```
行間: 1.5（24px at 16px font）
段落間: 3.0（48px at 16px font） → 実装では 1.5rem-2rem

実装:
<div class="space-y-6">  <!-- 24px: 行間の2倍ではなく、適度な間隔 -->
  <p class="leading-normal">第1段落...</p>
  <p class="leading-normal">第2段落...</p>
</div>
```

### 段落の先頭インデント vs 段落間隔

**2つのアプローチ**:

1. **段落間隔方式**（推奨 - Web標準）
   - 段落間に余白を設ける
   - 先頭インデントなし
   - スキャンしやすい

2. **先頭インデント方式**（印刷物）
   - 段落間隔なし
   - 先頭に1em程度インデント
   - Webでは非推奨

**このデザインシステムでは段落間隔方式を採用**

## テキスト配置（Alignment）

### 推奨する配置

```
推奨: 左揃え（Left-aligned）
  - 自然な読み始め位置
  - 予測可能な行頭
  - 視線の移動が一定

避ける: 両端揃え（Justified）
  - 単語間に不自然な空白
  - 可読性を損なう
  - 日本語では特に不適切
```

### 例外ケース

- **センター揃え**: 見出し、短いキャッチコピーのみ
- **右揃え**: 数値列、RTL言語

## 段落の最大幅とコンテナ

### コンテンツコンテナの設計

```
全画面幅のコンテナ:
<div class="w-full">
  
  中央配置、最大幅制限:
  <div class="max-w-7xl mx-auto px-4">
    
    さらに記事本文は狭く:
    <article class="max-w-2xl mx-auto">
      <p>本文...</p>
    </article>
  </div>
</div>
```

**原則**: 
- レイアウトコンテナ: 最大幅7xl（1280px）
- 記事コンテナ: 最大幅2xl（672px）または prose（65ch）

## コントラストと背景

### テキストと背景のコントラスト

可読性に直接影響するため、WCAG基準を厳守します。

**最小コントラスト比**:
- 通常テキスト（14-18px）: 4.5:1（AA）
- 大きなテキスト（18px以上）: 3:1（AA）

### 推奨する組み合わせ

```
良い例:
- 黒テキスト（#1a1a1a）on 白背景（#ffffff）: 15.3:1
- 濃いグレー（#4a4a4a）on 白背景: 8.6:1

避ける例:
- グレー（#9ca3af）on 白背景: 2.8:1 ← 基準未満
- 青（#60a5fa）on 白背景（小さいテキスト）: 2.9:1 ← 基準未満
```

→ 詳細は [Color Meaning](../color/color-meaning.md)、[Accessibility](../foundation/accessibility.md) 参照

## 行間（Line Height）の最適化

### コンテキスト別の行間

すでに [Typography Scale](./typography-scale.md) で定義されていますが、可読性の観点から補足：

**長文コンテンツ（記事等）**:
- 本文: 1.5-1.625（relaxed推奨）
- リード文: 1.625-1.75
- 理由: 長時間読むため、ゆったりした行間が疲労を軽減

**短文コンテンツ（UI、ラベル）**:
- ラベル: 1.5
- ボタン: 1-1.25
- 理由: 素早く読むため、行間は詰めて簡潔に

## リスト・箇条書きの可読性

### リスト項目間の余白

```
密なリスト（選択肢、メニュー等）:
<ul class="space-y-1">  <!-- 4px -->
  <li>項目1</li>
  <li>項目2</li>
</ul>

標準リスト:
<ul class="space-y-2">  <!-- 8px -->
  <li>項目1</li>
  <li>項目2</li>
</ul>

説明的リスト:
<ul class="space-y-4">  <!-- 16px -->
  <li>
    <strong>項目1</strong>
    <p>説明文...</p>
  </li>
  <li>
    <strong>項目2</strong>
    <p>説明文...</p>
  </li>
</ul>
```

### インデントとマーカー

```
ネストしたリスト:
<ul class="list-disc list-inside space-y-2">
  <li>レベル1
    <ul class="list-circle list-inside ml-6 mt-2 space-y-1">
      <li>レベル2</li>
    </ul>
  </li>
</ul>
```

## モバイルでの可読性

### フォントサイズ

- **最小16px**: モバイルブラウザのズーム防止
- 本文はデスクトップと同じ16px推奨

### パディング・マージン

```
モバイル:
- コンテナ: 16px (px-4)
- 段落間: 16px (space-y-4)

デスクトップ:
- コンテナ: 32px (px-8)
- 段落間: 24px (space-y-6)
```

### タップ領域

リンクやボタンは最小44px × 44pxを確保

→ 詳細は [Accessibility](../foundation/accessibility.md) 参照

## アクセシビリティ考慮

### ズーム対応

- **200%ズームテスト**: WCAG 2.0 AA基準
- rem単位の使用で対応
- 横スクロールが発生しないよう設計

### スクリーンリーダー

- 論理的な見出し階層（h1→h2→h3、スキップなし）
- 段落は`<p>`タグで明示的にマークアップ
- リストは`<ul>`/`<ol>`/`<li>`で構造化

### ディスレクシア対応

- 行間を1.5以上に設定（WCAG推奨）
- 段落間を明確に
- 両端揃えを避ける
- サンセリフフォント推奨

## 測定方法

### チェックリスト

- [ ] 行長が適切（英65ch、日35ch以内）
- [ ] 段落間が行間の1.5倍以上
- [ ] 本文行間が1.5以上
- [ ] テキストと背景のコントラスト比がWCAG AA基準
- [ ] 左揃え（両端揃えを使用していない）
- [ ] モバイルで本文16px以上
- [ ] 200%ズームで横スクロール不要
- [ ] 見出し階層が論理的

### 定量的評価

- **Lighthouse**: 可読性スコア
- **WAVE**: コントラスト比チェック
- **読了時間**: 平均ページ滞在時間（Analytics）
