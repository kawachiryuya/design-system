# マイクロコピー（Microcopy）

## メタ情報
- カテゴリ: コンテンツ原則 > Writing
- 適用範囲: Atom〜Molecule
- ステータス: Approved
- 最終更新: 2024-02-17

## 原則の定義

**UI内の小さなテキストが、ユーザーの迷いを取り除き、次のアクションへ自然に誘導する**

マイクロコピーとは、ボタンラベル・プレースホルダー・ヘルプテキスト・ツールチップなど、UI上の短いテキストのことです。文字数は少ないが、UXへの影響は大きい。

## ボタンラベル

### 動詞 + 目的語の形にする

```
✅ 良い例:
「保存する」「削除する」「送信する」「プランを選ぶ」

❌ 悪い例:
「OK」「はい」「送信」（動詞だけで目的語がない）
「クリックして次へ」（操作方法を説明しすぎ）
```

「OK」「はい」はモーダルのような文脈では許容されるが、
できる限り「何をするのか」を明示するラベルにする。

### 破壊的アクションは明確に

```
✅ 削除ダイアログのボタン:
[キャンセル]  [プロジェクトを削除]

❌:
[いいえ]  [はい]
→ どちらが削除か一瞬考える
```

### 状態変化を反映する

```
通常時:     [フォローする]
フォロー後:  [フォロー中]
ホバー時:   [フォロー解除]
```

## プレースホルダー

### 入力形式の例を示す

```
✅ 良い例:
メールアドレス
[例: user@example.com  ]

パスワード
[8文字以上              ]

✅ または何も書かない（ラベルが明確なら不要）
```

### プレースホルダーだけでラベルを代替しない

```
❌ 悪い例:
[メールアドレスを入力してください]
→ 入力後にラベルが消えて何を入力したか分からない

✅ 良い例:
メールアドレス         ← ラベル（消えない）
[例: user@example.com] ← プレースホルダー（補足）
```

→ [Forms](../../patterns/forms.md) 参照

## ヘルプテキスト

### いつ使うか

```
✅ 使う場面:
- 入力形式が分かりにくい（パスワード要件、電話番号形式）
- 入力内容が後から変更しにくい（ユーザー名など）
- セキュリティや利用規約に関わる補足

❌ 使わない場面:
- ラベルだけで十分な場合（「氏名」など）
- 自明な入力（郵便番号フィールドに「郵便番号を入力」）
```

### 書き方

```
✅ 良い例:
「半角英数字・記号を組み合わせた8文字以上」
「登録後は変更できません」
「公開プロフィールに表示されます」

❌ 悪い例:
「このフィールドはパスワードの入力に使用されます」（自明）
「入力してください」（何も情報を加えていない）
```

## ツールチップ

### 使うタイミング

```
✅ 向いている:
- アイコンのみのボタンの説明
- 略語・専門用語の説明
- スペースの都合でラベルを省略した場合

❌ 向いていない:
- 重要な情報（ツールチップは常に見えない）
- タッチデバイスでの主要なUI（ホバーできない）
```

### 書き方

```
✅ 短く・体言止め（20文字以内）:
「ファイルをダウンロード」
「フィルターを解除」
「共有リンクをコピー」

❌ 長い文章:
「このボタンをクリックすると、現在表示されているデータをCSV形式でダウンロードします」
```

## ラベル・見出し

### スキャンしやすくする

```
✅ 体言止め・名詞中心（メニュー・設定・見出し）:
「通知設定」「セキュリティ」「プランと請求」

✅ 動詞+目的語（ボタン・CTA）:
「パスワードを変更」「プランをアップグレード」

❌ 混在:
「通知の設定をする」← 見出しなのに動詞形
「セキュリティ」← ボタンなのに体言止め
```

## 空状態・オンボーディング

```
✅ 良い空状態コピー:
見出し: 「まだプロジェクトがありません」（状況の説明）
本文:   「プロジェクトを作成してチームで共有しましょう」（価値の提示）
CTA:    「最初のプロジェクトを作成」（具体的なアクション）

❌ 悪い空状態コピー:
見出し: 「データがありません」（何も情報を加えていない）
CTA:    「追加」（何を追加するか不明）
```

→ [Data Display](../../patterns/data-display.md) 参照

## チェックリスト

- [ ] ボタンラベルが「動詞 + 目的語」の形
- [ ] 破壊的アクションのボタンに明確なラベル
- [ ] プレースホルダーがラベルの代替になっていない
- [ ] ヘルプテキストが価値ある情報を追加している
- [ ] ツールチップが20文字以内
- [ ] 空状態に「状況の説明 + 価値の提示 + CTA」がある
