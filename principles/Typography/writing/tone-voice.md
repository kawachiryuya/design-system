# トーンとボイス（Tone & Voice）

## メタ情報

## 原則の定義

**ボイスは変えず、トーンを状況に合わせる**

ボイス（Voice）はブランドの個性であり、常に一定。トーン（Tone）は状況や感情に応じて変化する。人間が「同じ人格」を持ちながら、場面に応じて話し方を変えるのと同じです。

## ボイスの定義

このデザインシステムのボイスは以下の4つの軸で定義します。

### 1. 明快（Clear）

```
❌ 曖昧: 「操作を完了するために必要な情報が不足しています」
✅ 明快: 「名前を入力してください」
```

### 2. 誠実（Honest）

```
❌ 誇張: 「完璧な体験をお届けします！」
✅ 誠実: 「シンプルで使いやすいツールです」
```

### 3. 親しみやすい（Friendly）

```
❌ 冷たい: 「エラーが発生しました。再試行してください。」
✅ 親しみ: 「うまく保存できませんでした。もう一度試してみてください。」
```

### 4. 簡潔（Concise）

```
❌ 冗長: 「続行するには、以下のボタンをクリックしてください。」
✅ 簡潔: 「続ける」
```

## トーンの調整

状況に応じてトーンを変える。ボイスの軸（明快・誠実・親しみ・簡潔）は変えない。

| 状況 | トーン | 例 |
|------|-------|-----|
| 通常操作 | 中立・効率的 | 「保存しました」 |
| 成功・完了 | 明るく前向き | 「送信完了です！」 |
| エラー | 落ち着いて、解決策を示す | 「保存できませんでした。再度お試しください。」 |
| 危険・警告 | 明確・直接的 | 「削除すると元に戻せません」 |
| 空状態・オンボーディング | 励まし・歓迎 | 「最初のプロジェクトを作ってみましょう」 |

## 人称・敬語の統一

### 人称

```
UIテキスト: 「あなた」は使わず、直接語りかける
✅ 「プロフィールを編集」
❌ 「あなたのプロフィールを編集してください」

所有格: 「あなたの」ではなく「マイ」または省略
✅ 「マイページ」「設定」
❌ 「あなたの設定」
```

### 敬語レベル

プロダクトの文脈に応じて統一する。どちらかに決めたら全体で一貫させる。

```
丁寧体（です・ます）: BtoBツール・公式ページ向け
  「保存しました」「送信してください」

カジュアル体: コンシューマー向け・フレンドリーなプロダクト向け
  「保存したよ」「送ってみよう」
```

→ [Consistency](../../foundation/consistency.md) 参照

## 能動態と受動態

```
✅ 能動態（推奨）:
「ファイルを削除しました」
「設定を保存してください」

❌ 受動態（避ける）:
「ファイルが削除されました」
「設定が保存される必要があります」
```

能動態の方が明快で責任の所在が分かりやすい。

## 避けるべき表現

### 専門用語・略語

```
❌ 「APIキーを設定してください」（ユーザーが非エンジニアの場合）
✅ 「連携用のキーを設定してください」

❌ 「キャッシュをクリアしてください」
✅ 「データを更新してください」または「再読み込みしてください」
```

### 脅しや罪悪感を与える表現（ダークパターン）

```
❌ 「退会すると、すべてのデータが永久に失われます！」（過度な脅し）
✅ 「退会後はデータにアクセスできなくなります」（事実のみ）

❌ 「本当に退会しますか？」ボタン: [いいえ、続ける] [はい、退会する]
✅ [キャンセル] [退会する]
```

### 曖昧な指示語

```
❌ 「こちらをクリックしてください」
✅ 「詳細を確認する」「プランを選択する」
```

- [ ] 能動態を使っている
